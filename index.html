~2005

The Code Writer

What the role looked like

  • Work assigned as small functional units
  • Write code → hand off
  • Limited visibility beyond own module

Responsibilities

  • Implement business logic
  • Fix bugs in assigned area
  • Write minimal unit tests

What they did NOT do

  • No deployment responsibility
  • No infra awareness
  • No production ownership
~2005 · Example

"Build a login module"

Developer would:

  • Write authentication logic
  • Maybe connect to DB
  • Hand off to QA
👉 Focus: code correctness

That's it. QA, deployment, production - someone else's problem.

2012–2018

Full-Stack + DevOps-Aware

This is where DevOps culture hit.

What changed

  • CI/CD pipelines introduced
  • Cloud started becoming mainstream
  • "You build it, you run it" mindset

New responsibilities

  • Write code + deploy it
  • Understand pipelines (builds, releases)
  • Handle basic infra configs
2012–2018 · Example

"Build and deploy the login service"

Developer would:

  • Write the API
  • Create a CI pipeline
  • Deploy to cloud (VM / early PaaS)
  • Debug deployment issues
👉 Focus: delivery + deployment

Ownership expanded beyond code for the first time.

2018–2023

SRE-Influenced Developer

Reliability became first-class.

What changed

  • Microservices exploded
  • Distributed systems complexity
  • Observability became critical

New responsibilities

  • Monitoring + alerting
  • Performance tuning
  • Incident response

New concepts

  • SLIs / SLOs
  • Error budgets
  • Resilience engineering
2018–2023 · Example

"Own the login service in production"

Developer would:

  • Add logging + metrics
  • Set up alerts
  • Respond to outages
  • Improve latency + reliability
👉 Focus: reliability + production health

Now they own production health end-to-end.

GitHub Copilot - Token-based Billing

Why this change

Usage-value alignment
  • Customers pay only for what they use
  • Clear visibility into AI consumption and cost drivers
  • Eliminates shelfware and overprovisioning
Pricing that scales with capability
  • More powerful models and agentic workflows consume more compute
  • Flat-rate pricing breaks down as usage and complexity grow
  • Token billing ties price directly to compute consumed
Industry-standard alignment
  • All major AI platforms price on tokens (OpenAI, Anthropic, Google, Cursor)
  • Reduces friction when customers compare or consolidate vendors

What this means for the customer

🤝 Pooled entitlements and budgeting controls
  • Share usage across teams (heavy + light users)
  • Enterprise and user-level budget controls
🚀 Access to ongoing innovation
  • Pricing scales with models and workflows
  • No upgrades or delays to access new capabilities
👁 Consumption insight
  • Visibility into usage by user and model
  • See actual AI cost using industry standard token rates

Tokens & Billing Explained

Tokens are units of AI processing (input, output, and context)

📥

Input Tokens

  • What you send:
  • Prompts and new context
  • Can grow with large files/codebases
📤

Output Tokens

  • What you get:
  • AI-generated responses
  • Typically, the highest compute cost
♻️

Cache Tokens

  • What's reused:
  • Context from previous interactions
  • Improves speed and efficiency
Billable tokens = Input tokens + Output tokens + Cache tokens
What this means:

Every interaction consumes tokens based on input, output, and context. Larger prompts, longer responses, or more context result in more tokens processed and more credits consumed.
